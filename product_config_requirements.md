## 三、第2页产品选择逻辑

### 3.1 可选产品（根据容积率）

| 容积率 | 可选产品 |
|:---|:---|
| 1.0 - 1.5 | 轻钢厂房、分栋厂房、配套宿舍、配套楼 |
| 1.5 - 2.0 | 分栋厂房、分层厂房、配套宿舍、配套楼 |
| > 2.0 | 分栋厂房、分层厂房、产业大厦、配套宿舍、配套楼 |

- 若第1页配套用房占比 > 0，则必须勾选**配套宿舍**或**配套楼**或**两者**
- 否则提示：「配套用房未选择」

| 容积率 | 处理逻辑 |
|:---|:---|
| ≤ 2.0 | 无产业大厦选项，提示「研发办公用房将与厂房合并配置，用户可根据需求在得到结果后自行手动分配研发办公用房」 |
| > 2.0 | 产业大厦为**必选项**，未勾选提示「研发办公用房未选择」 |

## 五、配置算法规则

### 5.1 控制指标

| 指标 | 目标值 | 允许误差 |
|:---|:---|:---:|
| 建筑密度 | 容积率对应密度（45%/42%/40%） | ±5% |
| 容积率 | 用户输入值 | ±0.05%（万分之五） |
| 计容面积 | 地上建筑面积×容积率 | ±0.05% |

### 5.2 配置优先级

1. **优先低层产品**：轻钢 → 分栋 → 分层 → 产业大厦
2. **必须配齐**：用户勾选的所有产品类型都必须出现
3. **数量下限**：轻钢/分栋/分层 ≥ 2栋（如勾选）
4. **数量上限**：产业大厦/配套宿舍/配套楼 ≤ 1栋（如勾选）
5. **占比控制**：配套用房、研发办公用房不超用户输入占比

### 5.3 计算逻辑

```
总基底面积 = 各产品(单栋基底 × 栋数)之和
建筑密度 = 总基底面积 / 规划用地面积
目标密度 = f(容积率) → 1.0-1.5:45% | 1.5-2.0:42% | >2.0:40%

总建面 = 各产品(单栋建面 × 栋数)之和
总计容 = 各产品(单栋计容 × 栋数)之和
容积率 = 总计容 / 规划用地面积

约束条件：
  |建筑密度 - 目标密度| ≤ 5%
  |容积率 - 目标容积率| ≤ 0.05%
  各产品栋数满足上下限
```

## 七、已确认规则补充

### 7.1 产业大厦层数反推（问题1确认）

- 层数**不固定**为10或12层
- 根据研发办公用房面积反推：研发面积 = 地上建筑面积 × 研发办公用房占比
- 反推逻辑：
  1. 参考用户选择的**较高分层厂房**的建筑高度
  2. 用产业大厦层高配置（首层6.6m，标准层4.5m）→ 估算层数
  3. 单层面积 = 研发面积 / 估算层数
  4. 单层面积 < 1200 → 固定1200为基底，能做几层做几层
  5. 单层面积 > 2400 → 固定2400为基底，能做几层做几层
  6. 超限时 → 先降标准层到4.2m → 仍不行则增加单层面积
  7. 面积输出尽量是100的倍数；差异大（配置后研发占比与用户输入相差0.2%）时用50的倍数
  8. 尽量靠近用户选择的面积选项

### 7.2 分层厂房1000/1200可独可双（问题2确认）

- 可设置1000独栋，也可设置1000+1000双拼
- **优先双拼**

### 7.3 配套宿舍层数反推（问题3确认）

**仅选配套宿舍时：**
1. 配套用房总面积 = 地上建筑面积 × 配套用房占比
2. 参考用户选择的**较低分层厂房**的建筑高度
3. 宿舍层高：首层4.8m，标准层3.6m → 估算层数
4. 单层面积 = 配套用房面积 / 估算层数
5. 单层面积 < 800 → 固定800
6. 单层面积 > 1200 → 固定1200
7. 超限时 → 先降标准层到3.3m → 仍不行则增加单层面积

**同时选配套宿舍+配套楼时：**
1. 面积比 2:1（宿舍:配套楼）
2. 配套楼面积 < 1200 → 固定1200，剩余给宿舍
3. 配套楼面积 > 2400 → 固定2400，剩余给宿舍
4. 宿舍再按"仅选配套宿舍"逻辑处理

### 7.4 配置算法迭代策略（问题4确认）

- 密度超了 → **减少低层，增加高层**
- 示例：2.0容积率时，减少分栋厂房，增加分层厂房

### 7.5 分栋厂房3.5层（问题5确认）

- 本质还是4层，第四层只算半层面积
- **层高分配和4层分栋厂房完全一样**
- 单栋建筑面积 = 基底面积 × 3.5

### 7.6 产业大厦在C65下（问题6确认）

- 仅在**上海+C65+容积率>2.0**时出现
- 研发办公用房面积 = 地上建筑面积 × 研发办公用房占比
- 研发办公用房即为产业大厦
- 按7.1规则反推层数和基底
